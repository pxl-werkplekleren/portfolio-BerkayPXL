Ontwikkeling

Mijn Keuze

Waarom Programmeren?

Ik heb voor deze opleiding gekozen omdat de interesse voor computers in de familie zit: ik ben er dus eigenlijk mee opgegroeid. Daarnaast vind ik het geweldig om dingen op te bouwen vanuit het niets. Coderen voelt voor mij als een logische puzzel: je begint met een probleem en zoekt stap voor stap naar de oplossing. Het geeft een kick als je code na hard werken foutloos draait. Net als in de fitness zie je bij programmeren dat discipline loont: als je blijft oefenen, word je steeds beter.

Waarom dit beroep?

Ik denk dat een job in de IT goed bij mij past omdat het een sector is met veel toekomst en werkzekerheid. De technologie staat nooit stil, wat betekent dat je je leven lang moet blijven bijleren. Die uitdaging spreekt me aan, want ik wil niet stilstaan. Daarnaast is software ontwikkelen vaak teamwerk. Door mijn ervaring met zaalvoetbal weet ik hoe belangrijk communicatie en samenwerken is om een doel te bereiken. Ik wil graag werken in een omgeving waar we samen naar oplossingen zoeken en waar resultaat telt.

Rol: Programmeur

Waar sta ik nu?

  • Huidige status: Ik ben momenteel een startende programmeur. Ik begrijp de basis van C# (zoals variabelen en loops) en kan eenvoudige websites bouwen met HTML, maar ik heb nog hulp nodig bij complexe oefeningen.
  • Sterk punt: Doorzettingsvermogen. Net als in de fitness geef ik niet op als het zwaar wordt. Als mijn code niet werkt, blijf ik zoeken tot ik de oplossing heb gevonden.
  • Werkpunt: Nauwkeurigheid. Ik wil soms te snel resultaat zien (zoals bij zaalvoetbal snel naar het doel), waardoor ik soms kleine typfoutjes maak in de code die voor errors zorgen.

3 Activiteiten

  1. Code schrijven: Ik heb zelfstandig code geschreven in Visual Studio om een werkende rekenmachine te maken. Hierbij moest ik de juiste commando's gebruiken om het programma te laten rekenen.
  2. Debugging: Tijdens het programmeren kreeg ik rode lijntjes (errors). Ik heb de foutmeldingen gelezen en mijn code aangepast totdat het programma weer werkte zonder vast te lopen.
  3. Logisch nadenken: Toen ik klaar was (of als ik vastzat), heb ik alles stap voor stap opnieuw overlopen. Ik ging regel voor regel door mijn eigen code heen om te kijken of de volgorde logisch was en of ik niets over het hoofd had gezien.

Reflectie

AIM Reflectie op WPL1

De start (Intro):
Mijn eerste semester Werkplekleren was een periode van veel nieuwe indrukken. Omdat IT in mijn familie zit, had ik er al wel een beeld bij, maar het zelf doen is toch anders. Het was soms pittig om de juiste logica te vinden, maar ik heb gemerkt dat ik door oefening snel stappen vooruit zet.

Wat ging goed (Positief):
Het leukste vond ik het moment waarop een programma eindelijk werkt. Dat geeft dezelfde voldoening als een doel bereiken in de fitness na hard trainen. Vooral bij de C# opdrachten vond ik het geweldig om te zien hoe een paar regels tekst een werkende applicatie kunnen vormen.

Wat was lastig (Minder leuk):
Waar ik moeite mee had, was het "debuggen" van kleine foutjes. Ik kan soms ongeduldig zijn als ik niet direct zie waar de fout zit. Het is frustrerend als je code niet werkt door één vergeten puntkomma of haakje, terwijl je logica wel klopt.

Wat heb ik geleerd (Inzichten):
Mijn belangrijkste inzicht is dat programmeren niet alleen typen is, maar vooral nadenken. In het begin begon ik meteen te typen, maar ik heb geleerd dat het beter is om eerst rustig te kijken wat er moet gebeuren. Structuur is belangrijker dan snelheid.

Mijn trots (Succes):
Ik ben het meest trots op dit portfolio. Het laat zien dat ik niet alleen de code begrijp, maar het ook netjes kan presenteren. Het bewijst voor mij dat ik de juiste studiekeuze heb gemaakt.

Volgende WPL (Kernkwadranten)

Mijn kwaliteit: Doorzettingsvermogen. Ik geef niet snel op.

Mijn valkuil: Soms wil ik te snel resultaat zien (ongeduld), waardoor ik slordigheidsfoutjes maak.

Mijn leerdoel: Voor het volgende semester wil ik werken aan mijn nauwkeurigheid. Ik wil leren om eerst rustig een plan te maken voordat ik begin met bouwen, zodat ik minder tijd kwijt ben aan het herstellen van kleine fouten.

X-Factor

Samenwerken (Empathisch/Sociaal):
Door mijn ervaring met zaalvoetbal weet ik dat je alleen wint als je samenwerkt. Tijdens de lessen heb ik geprobeerd mijn medestudenten te helpen waar ik kon en feedback te vragen aan anderen.

Passie (Gedreven):
Ik ben gemotiveerd om de beste versie van mezelf te worden, zowel in de sport als in mijn studies. Ik stop niet als de les voorbij is, maar probeer thuis verder te oefenen.
